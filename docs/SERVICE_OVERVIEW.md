# メンエスインデクサ サービス概要書

## 1. サービスコンセプト

**「"確認"で終わるサイトから、"発見"が始まるサイトへ」**

ユーザーは「セラピスト名 + 口コミ」で検索し、既存サイト（ME）にたどり着く。
しかしユーザーの本当の欲求は「この子で大丈夫か？」という確認だけではない。
**「もっと良い子がいるなら、そっちを知りたい」** という発見欲求がある。

メンエスインデクサは、この「発見」のニーズに応える。
口コミ投稿から構造化データを収集し、**「同じ系統 × 同じエリア × 高評価」のセラピストを提案（レコメンド）する**、次世代のプラットフォームを目指す。

---

## 2. 競合分析（ME）

### MEができていること（強み）
- **エリア検索**: 都道府県 → 地区 → 店舗 → セラピスト の階層が整備されている。
- **詳細な口コミ評価**: 8項目（ルックス、スタイル、会話、衣装、施術、鼠径部、密着、寛容）+ 100点満点。
- **コミュニティ機能**: 掲示板、チャット、BHリスト（過激口コミ）、感謝バッジなど。
- **豊富なコンテンツ**: 約20万人分のセラピストデータ・口コミが蓄積されている。

### MEができていないこと（弱点 = 勝機）
| MEの限界 | 具体的な問題 |
| :--- | :--- |
| **名前を知らないと探せない** | 検索は「サロン名」「セラピスト名」のテキスト入力のみ。「ギャル系で巨乳の子」といった属性検索は不可能。 |
| **他店舗へのレコメンドがない** | 店舗ページには同じ店のセラピストしか表示されない。「このエリアで同じタイプの高評価セラピスト」という提案がゼロ。 |
| **口コミデータが検索に活かされていない** | ルックス、スタイル、密着度などの評価項目は表示されるが、それで絞り込む機能がない。宝の持ち腐れ。 |
| **投稿ハードルが高い** | 自由記述の長文が前提。何を書けばいいかわからず、面倒。 |
| **有料会員の投稿動機がない** | 月額4,980円払えば全部見れるので、自ら情報発信する理由がない。情報の鮮度が落ちる。 |

---

## 3. メンエスインデクサの解決策

### A. 「タイプ検索」で発見体験を提供
- **機能**: 口コミ投稿時に「見た目のタイプ」「スタイル」「アピールタグ」を選択式で収集。
- **効果**: 「ギャル × 巨乳 × 渋谷エリア × 80点以上」といった横断検索が可能に。
- **体験**: ユーザーは「名前を知らなくても、好みのタイプで探せる」。

### B. レコメンドエンジン（Tinderライク）
- **機能**: セラピスト詳細ページに「同じ系統でもっと評価高い子」を自動表示。
- **効果**: 「アロマモアのAさんを見たけど、同じギャル系でアロマエメラルドのBさんの方が評価高いよ」と提案。
- **体験**: ユーザーは「知らなかった最高の子」に出会える。この人好きそうな人はこれも好きそうだな、という提案が可能に。

### C. タップ式投稿フォーム（1問1答形式）
- **機能**: Tinder的なUI。タイプ・スタイル・サービスレベル・パラメータをタップ/スライダーで選択。
- **効果**: 投稿ハードルが劇的に下がり、かつ構造化データが集まる。
- **体験**: スマホで1〜2分で投稿完了。ゲーム感覚でサクサク進む。

### D. Review to Earn（投稿割引モデル）
- **機能**: 有料会員が口コミを投稿するたびに、翌月の会費を¥1,000割引（最大¥4,000/月）。
- **効果**: 有料会員にも投稿動機が生まれ、情報の鮮度が維持される。サイト側は常に最新の口コミ（サイトの資産）を収集し続けられる。
- **体験**: 月4回投稿すれば、実質¥980でサービス利用可能。

## 4. ビジネスモデル

### 会員体系

| 会員種別 | 月額 | 閲覧権限 | 投稿特典 |
| :--- | :--- | :--- | :--- |
| **無料会員** | ¥0 | **投稿1件ごとに24時間の閲覧権限を付与** | - |
| **スタンダード会員** | ¥4,980 | 無制限 | 投稿1件ごとに翌月¥1,000割引（最大¥4,000） |
| **VIP会員** | ¥14,980 | 無制限 + 詳細フィルタ | 同上 |

### VIP会員限定機能
- **サービスレベルでのフィルタリング**: 「健全〜過激」の4段階で絞り込み可能。ユーザーが最も知りたがっている「具体的なサービス内容（本番の有無など）」で検索できる。
- **裏掲示板アクセス**: よりディープな情報交換の場。

### 収益モデル
1. **サブスクリプション**: スタンダード会員・VIP会員からの月額課金。
2. **店舗課金（将来）**: 店舗ページのリッチ化、求人機能、予約システム連携など。

## 5. 成長戦略（フェーズ分け）

### Phase 1: 初期データ投入 & SEO流入
1. 既存データベース（Supabase）のセラピスト情報（約20万人）と口コミデータを投入。
2. **「セラピスト名 + 口コミ」という検索キーワードの強さを活かし、短期間でのアクセス増を狙う。**
3. 初期は競合と同等のコンテンツ量で「確認」ニーズを満たしつつ、流入を確保。

### Phase 2: 独自データ蓄積 & 差別化
1. 流入したユーザーに「タップ式投稿」をさせ、構造化データを蓄積。
2. タイプ検索・レコメンドの精度が上がり、**「ここでしかできない発見体験」** が生まれる。
3. サイトのオリジナリティと信頼性が高まる。

### Phase 3: コミュニティ形成 & 定着
1. 掲示板、チャット、お気に入り共有などのコミュニティ機能を追加。
2. 「ここなら好みの新しい子が見つかる」という体験でリピート・定着。

## 7. UI/UX方針

### デザイン基本方針
- **PCファースト + レスポンシブ**: MEベースの情報量を維持しつつ、スマホでも見やすく。
- **MEに慣れたユーザーが迷わない**: 基本的なページ構成（トップ → エリア → 店舗 → セラピスト）は踏襲。
- **「発見」を際立たせるUI**: タイプ検索、レコメンド欄、レーダーチャートなど独自要素を追加。

### 独自UI要素
- **「タイプから探す」グリッド**: トップページに6つのタイプカードを配置。
- **レーダーチャート**: セラピスト詳細ページにパラメータを可視化。
- **タイプ成分分析（円グラフ）**: 「ギャル80%、アイドル20%」のような分布表示。
- **レコメンド欄**: 「同じタイプでもっと高評価の子」を自動表示。
- **口コミブラー（ぼかし）**: 未投稿ユーザーには口コミ本文をぼかして表示。投稿CTAを促す。

## 8. まとめ：なぜMEを超えられるのか

### MEの限界
**「名前を知っている人の評判を"確認"するサイト」** でしかない。

### メンエスインデクサの強み
**「好みのタイプから知らない最高の子を"発見"するサイト」** になれる。

### 差別化の4つの柱

| # | 戦略 | 具体策 |
| :--- | :--- | :--- |
| 1 | **検索意図の上位概念を解決** | MEは「確認」で終わる。うちは「発見」まで連れて行く。 |
| 2 | **データを活かす設計** | 口コミデータを「表示」だけでなく「検索・レコメンド」に活用。 |
| 3 | **投稿者にメリットを還元** | 無料会員には24時間閲覧権、有料会員には月額割引。情報の鮮度が保たれる。 |
| 4 | **投稿ハードルを極限まで下げる** | タップ式で1分投稿。データの量と質を両立。 |
