# メンエスインデクサ サービス概要書

## 1. サービスコンセプト

**「"確認"で終わるサイトから、"発見"が始まるサイトへ」**

ユーザーは「セラピスト名 + 口コミ」で検索し、既存サイト（ME）にたどり着く。
しかしユーザーの本当の欲求は「この子で大丈夫か？」という確認だけではない。
**「もっと良い子がいるなら、そっちを知りたい」** という発見欲求がある。

メンエスインデクサは、この「発見」のニーズに応える。
口コミ投稿から構造化データを収集し、**「同じ系統 × 同じエリア × 高評価」のセラピストを提案（レコメンド）する**、次世代のプラットフォームを目指す。

---

## 2. 競合分析（ME）

### MEができていること（強み）
- **エリア検索**: 都道府県 → 地区 → 店舗 → セラピスト の階層が整備されている。
- **詳細な口コミ評価**: 8項目（ルックス、スタイル、会話、衣装、施術、鼠径部、密着、寛容）+ 100点満点。
- **コミュニティ機能**: 掲示板、チャット、BHリスト（過激口コミ）、感謝バッジなど。
- **豊富なコンテンツ**: 約20万人分のセラピストデータ・口コミが蓄積されている。

### MEができていないこと（弱点 = 勝機）
| MEの限界 | 具体的な問題 |
| :--- | :--- |
| **名前を知らないと探せない** | 検索は「サロン名」「セラピスト名」のテキスト入力のみ。「ギャル系で巨乳の子」といった属性検索は不可能。 |
| **他店舗へのレコメンドがない** | 店舗ページには同じ店のセラピストしか表示されない。「このエリアで同じタイプの高評価セラピスト」という提案がゼロ。 |
| **口コミデータが検索に活かされていない** | ルックス、スタイル、密着度などの評価項目は表示されるが、それで絞り込む機能がない。宝の持ち腐れ。 |
| **投稿ハードルが高い** | 自由記述の長文が前提。何を書けばいいかわからず、面倒。 |
| **有料会員の投稿動機がない** | 月額4,980円払えば全部見れるので、自ら情報発信する理由がない。情報の鮮度が落ちる。 |

### MEの構造的弱点（追随できない理由）
| 弱点 | 理由 | うちの優位性 |
| :--- | :--- | :--- |
| 口コミデータが非構造化 | 5年前の設計。今から変えられない | 最初から構造化設計 |
| 有料会員が投稿しない | お金払った瞬間に投稿する意味がなくなる | Review to Earnで投稿動機を維持 |
| 無料ユーザーは「仕方なく」投稿 | ハードルが高い長文フォーム | タップ式で楽に投稿 |
| レコメンド機能がない | データがないから作れない | 構造化データでレコメンド可能 |

## 3. メンエスインデクサの解決策

### A. 「セラピスト検索」がメイン機能
- **機能**: タイプ、スタイル、エリア、点数、SKR/HRで絞り込み検索。
- **効果**: 「ギャル × 巨乳 × 渋谷エリア × 80点以上」といった横断検索が可能に。
- **体験**: ユーザーは「名前を知らなくても、好みのタイプで探せる」。
- **差別化**: SKR/HRフィルターは会員種別により段階的に解放。

### B. レコメンドエンジン（口コミベース）
- **機能**: ユーザーの閲覧履歴に含まれるセラピストのタグを分析し、同じタイプ × 近いエリア × 高評価のセラピストを提案。
- **ポイント**: セラピスト単位ではなく **口コミ単位** でレコメンド。
- **差別化**: SKR/HRありのレコメンドは会員種別により段階的に解放。

### C. タップ式投稿フォーム（1問1答形式）
- **機能**: Tinder的なUI。タイプ・スタイル・サービスレベル・パラメータをタップ/スライダーで選択。
- **効果**: 投稿ハードルが劇的に下がり、かつ構造化データが集まる。
- **体験**: スマホで1〜2分で投稿完了。ゲーム感覚でサクサク進む。

### D. Review to Earn（投稿割引モデル）
- **機能**: 有料会員が口コミを投稿するたびに、翌月の会費を¥1,000割引（最大¥4,000/月）。
- **効果**: 有料会員にも投稿動機が生まれ、情報の鮮度が維持される。

## 6. 口コミ投稿フォーム設計

### 設計思想
- **ユーザーにとって**: 楽に投稿できる（タップ式、分割質問）
- **システムにとって**: 高品質な構造化データが集まる（選択式で正規化）
- **Win-Win**: ユーザーの負担を減らしながら、レコメンドの精度を上げる

### 入力項目（詳細設計）

**Step 0: 対象選択**
- 店舗名（DBからサジェスト）
- セラピスト名（DBからサジェスト or 新規追加）

**Step 1: 見た目のタイプ（必須・単一選択）**

**Step 2: スタイル（必須・単一選択）**

- 普通
- グラマー（巨乳）

**Step 3: サービス満足度（必須・単一選択）**

## 7. UI/UX方針

### ナビゲーション構成

**ヘッダー（左から順に）:**
1. 🔍 セラピスト検索（メイン機能）
2. 📍 エリア検索
3. 🏆 ランキング
4. 📋 掲示板
5. ✏️ 口コミ投稿（ピンクボタン）
6. ログイン / 新規登録

**マイページサイドバー:**
1. 📊 ダッシュボード
2. 🔍 セラピスト検索
3. 📝 投稿した口コミ
4. ❤️ お気に入り
5. 🔗 リスト共有
6. 💬 メッセージ
7. 📋 掲示板
8. 🔥 SKR/HRリスト（🔒ロックマーク）
9. 🔔 通知
10. ⚙️ 設定

### デザイン基本方針
- **PCファースト + レスポンシブ**: MEベースの情報量を維持しつつ、スマホでも見やすく。
- **MEに慣れたユーザーが迷わない**: 基本的なページ構成は踏襲。
- **「発見」を際立たせるUI**: タイプ検索、レコメンド欄、レーダーチャートなど独自要素を追加。

### 独自UI要素
- **セラピスト検索ページ**: フィルター + 検索結果 + レコメンド
- **レーダーチャート**: セラピスト詳細ページにパラメータを可視化。
- **タイプ成分分析（円グラフ）**: 「ギャル80%、アイドル20%」のような分布表示。
- **レコメンド欄**: 「同じタイプでもっと高評価の子」を自動表示。
- **口コミブラー（ぼかし）**: 未投稿ユーザーには口コミ本文をぼかして表示。

## 10. 技術課題（要検討）

### 課題1: セラピスト在籍情報の鮮度

**問題:**
- MEのスクレイピングデータは「ある時点」のスナップショット
- 実際は毎日入店・退店がある
- 本家サロンをスクレイピングして最新情報を取る必要がある

**解決案: LLMベースの汎用スクレイパー**
```
1. サロンの在籍ページURLをfetch
2. Claude APIにHTMLを渡す
3. 「セラピスト名とURLのリスト」を抽出させる
4. セレクタを事前定義する必要なし
```

### 課題2: データの名寄せ

**解決案:**
- 既存の `therapist_url`（本家セラピストページURL）でマッチング
- 新規: DBにない → 新規登録
- 退店: DBにあるが本家に存在しない → status = "retired"

### 現在のSupabaseスキーマ

**salon_db:**
- salon_id, salon_name, area_id, area_name
- salon_url ← 本家サロンのURL
- me_salon_url, me_therapistlist_url

**therapist_db:**
- therapist_id, salon_id, area_id
- therapist_name, therapist_age, three_sizes
- therapist_url ← 本家セラピストページのURL
- me_therapist_url
- reviews_data, review_count, latest_date

## 11. まとめ：なぜMEを超えられるのか

### MEの限界
**「名前を知っている人の評判を"確認"するサイト」** でしかない。

### メンエスインデクサの強み
**「好みのタイプから知らない最高の子を"発見"するサイト」** になれる。

### 差別化の5つの柱

| # | 戦略 | 具体策 |
| :--- | :--- | :--- |
| 1 | **セラピスト検索がメイン機能** | タイプ×スタイル×エリア×点数×SKR/HRで絞り込み |
| 2 | **段階的な機能解放** | 無料→スタンダード→VIPで課金動機を明確に |
| 3 | **データを活かす設計** | 口コミデータを「表示」だけでなく「検索・レコメンド」に活用 |
| 4 | **投稿者にメリットを還元** | 無料会員には期間限定閲覧権、有料会員には月額割引 |
| 5 | **投稿ハードルを極限まで下げる** | タップ式で1分投稿。データの量と質を両立 |

---

**「アロマモアのAさんを見に来たユーザーが、セラピスト検索で同じギャル系×SKRありで絞り込み、アロマエメラルドのBさんの方がもっと良いと知って予約する」**

——この体験ができるのは、メンエスインデクサだけ。
