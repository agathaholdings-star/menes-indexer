## 核心的な課題: 1キーワード = 1ページ

### 要件
「恵比寿 メンズエステ」と「中目黒 メンズエステ」は別の検索キーワードなので、別のページが必要。

### 問題点
1. **esthe-rankingでは「恵比寿・中目黒・代官山」が1ページ**
   - データ取得は1ページでOKだが、うちのサイトでは3ページに分ける必要がある
   - サロンデータの振り分けが必要（同じサロンが3ページに出る or 駅で振り分け）

2. **「博多」に天神・中洲が含まれている問題**
   - esthe-rankingの163エリア名には「天神」「中洲」が含まれていない
   - 駅別サブページ（`/hakata/tenjin-station/all/`）からは取れる
   - MEでは最初から「天神・中州川端」として独立

3. **粒度のバランス**
   - 駅ベース（626件）は細かすぎる（太宰府、櫛田神社前レベルは不要）
   - 163そのままだと天神・中洲などの重要エリアが漏れる
   - ahrefsの検索データもプラン制限で2,500件までしか見えず、完全な検索ボリュームは不明

## 確定した方針（2026-02-07）

### 考え方

エリアページ設計で大事なのは**データソースが実在すること**。

- ahrefsで検索ボリュームがあっても、スクレイピング元にデータがなければページは作れない
- 逆に、スクレイピング元のエリア名をそのまま使うだけだと、検索キーワードに合わない
- **「データが取れる」×「検索需要がある」の積集合**がうちのエリアページになる

### データソースの全体像

esthe-ranking.jpの構造：
```
/ebisu/                          ← 163個（トップレベル、全サロン一覧）
  /ebisu/ebisu-station/all/      ← 569個（駅別フィルター、駅周辺のサロンだけ）
  /ebisu/nakameguro-station/all/ ← これも独立したデータソースとして使える
```

- **163ページ = MAXではない**
- 163の下にある駅別サブページ569件も、それぞれ独立したサロンデータを持っている
- 地方で駅別がないエリア57件はトップレベルページを直接使う
- → **データソースとしてのMAXは 569 + 57 = 626件**

### エリアページ決定のフロー

```
Step 1: esthe-rankingの全データソースをリストアップ
        （163エリア + 569駅別サブ = 全スクレイピング可能URL）
        ↓
Step 2: 各データソースのエリア名を抽出
        （恵比寿駅 → 恵比寿、天神駅 → 天神、等）
        ↓
Step 3: 検索需要でフィルタリング
        （ME・メンマガに存在する / ahrefsでボリュームがある → 採用）
        （どこにも存在しない小駅 → 不採用）
        ↓
Step 4: 採用したエリア名 = うちのエリアページ
        （各ページのサロンデータは、対応するesthe-rankingのURLから取得）
```

## SEOカニバリ解消（2026-02-08 更新）

### 問題

975件のうち、親エリア・駅・市が同じSEOキーワードを持つケースが多数ある。
例: `/ebisu/` と `/ebisu/ebisu-station/all/` が両方「恵比寿 メンズエステ」。

### SEOキーワード抽出バグ修正（2026-02-08）

**旧ルール**: 「先頭が県名と同じなら2番目を使う」
**問題**: 県庁所在地=県名のケースで誤作動

| parent_name | 旧KW（バグ） | 新KW（修正後） |
|-------------|-------------|---------------|
| 奈良・生駒 | 生駒 | **奈良** |
| 広島・福山 | 福山 | **広島** |
| 熊本・玉名 | 玉名 | **熊本** |
| 鹿児島・霧島 | 霧島 | **鹿児島** |
| 山形・米沢 | 米沢 | **山形** |

**修正ルール**: 「先頭=県名」かつ「その県に親エリアが1つだけ」→ 先頭をそのまま使う

### 解決方針

**原則: 1キーワード = 1ページ。全ページを「エリアページ」として統一。**

- 親-駅の重複 → 親吸収（駅ページは作らない）
- 親-市の重複 → 親吸収（市ページは作らない）
- 駅-市の重複 → station優先
- 別府県同名9組 → URL構造 `/area/{prefecture}/` で自然に解消
- グループ中間ページは**作らない**
- 都道府県ページがハブ。同グループのエリアは「近隣エリア」として相互リンク

### 具体例

**恵比寿グループ:**
```
恵比寿 メンズエステ  → data: /ebisu/（親吸収）
中目黒 メンズエステ  → data: /ebisu/nakameguro-station/all/（駅、unique）
代官山 メンズエステ  → data: /ebisu/daikanyama-station/all/（駅、unique）
広尾 メンズエステ    → data: /ebisu/hiroo-station/all/（駅、unique）
```

**地方エリア（city追加で新規カバー）:**
```
旭川 メンズエステ    → data: /sapporo/asahikawa-city/all/（市、unique）
函館 メンズエステ    → data: /sapporo/hakodate-city/all/（市、unique）
丸亀 メンズエステ    → data: /kagawa/marugame-city/all/（市、unique）
呉 メンズエステ      → data: /hiroshima/kure-city/all/（市、unique）
```
