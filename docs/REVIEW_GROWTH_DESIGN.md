# 口コミ最大化のためのサービス設計検討

> 目的: メンエス口コミサイトで口コミ数1位を取る
> 前提: 稼げるサイトではなく「口コミが集まるサイト」を最優先で設計する
> 収益は口コミループの副産物として自然に生まれる

---

## クレジット制の最新仕様

> **最新の確定仕様は `memory/launch-strategy.md` を参照**
> 初期クレジット0、口コミ承認後5（スクショ付き10）、7日失効

### Standard 段階的機能解放

| 今月の投稿数 | 解放される機能 |
|-------------|--------------|
| 0本 | 口コミ全文読み放題 |
| 1本 | + 発見検索 |
| 2本 | + セラピスト分析 + 掲示板・DM |
| 3本 | VIP相当の全機能解放 |

---

## 「口コミ最大化」視点での課題

### 課題1: 無料ユーザーがクレジットゼロで離脱する

- 3クレジット使い切る → 「口コミ書いて」→ 面倒 → サイトを閉じる → 戻ってこない
- **戻る理由がない**（通知・リマインドなし）
- **最初の1本が重い**（コメント3問が心理的ハードル）
- **書く動機が弱い**（10クレジットの体感価値が見えにくい）

### 課題2: Standard会員が3本書いたら「完了」になる

- 月3本でVIP相当 → 4本目以降を書く理由がない
- 口コミが「月3本の作業」に感じ始める → 3ヶ月後に解約
- **書いた後の反応がない**（view_countだけ。参考になった未実装）

### 課題3: VIP会員は書く「必要」がない

- 全機能解放済み → 消費専門になりがち
- 口コミサイトとして最もお金を出すユーザーが書かない問題

### 課題4: 口コミの「読む体験」が弱い

- 読む体験が弱い → サイト滞在時間が短い → 「もっと見たい」が生まれない
- SNS的なフィードがない → 能動的に検索しないと新着口コミに出会えない

## 検討すべき機能アイデア

### 無料離脱者を呼び戻す

| ID | 機能 | 概要 | 目的 |
|----|------|------|------|
| A | お気に入り新着通知 | お気に入りセラピストに新口コミ → メール通知 | 戻る理由を作る |
| B | チラ見せ改善 | ブラー上に「SKR / 92点 / 距離感5」だけ表示 | 「読みたい」を生む |
| C | クレジット消費の可視化 | 「10クレジット = 10人分の詳細」を具体的に見せる | 書く動機を強化 |
| D | 初回投稿ボーナス | 初投稿で+20クレジット（通常10の倍） | 最初の1本を超えさせる |
| E | ライト口コミ | スコア+サービスレベルだけで3クレジット | 初投稿のハードルを壊す |
| F | 「最近行った？」リマインド | お気に入りエリア閲覧後にメール | サイトへの動線 |

### 有料会員の継続投稿

| ID | 機能 | 概要 | 目的 |
|----|------|------|------|
| G | 参考になった + 通知 | 口コミに「参考になった」→ 投稿者に通知 | 投稿ループの燃料 |
| H | 口コミタイムライン | お気に入りの新着がフィードで流れる | SKRを開く習慣 |
| I | レコメンド精度可視化 | 「口コミ8本で精度74%→あと3本で82%」 | 3本で終わらせない |
| J | 月間レポート | 今月の訪問・傾向・好みが近い人の推し | 月次の呼び戻し |
| K | ストリークボーナス | 3ヶ月連続3本 → 来月VIP機能を投稿0で解放 | 継続の動機 |
| L | VIPバッジ口コミ | VIPの口コミに専用マーク+上位表示 | VIPの投稿動機 |
| M | VIP限定深い分析 | 投稿5件以上で「隠れ名セラピスト」生成 | VIPでも書く理由 |

### 比較・発見（滞在時間向上）

| ID | 機能 | 概要 |
|----|------|------|
| N | セラピスト比較 | 2-3人を横並びレーダーチャート比較 |
| O | 口コミリクエスト | 口コミゼロの子に「口コミ募集中」ボタン |
| P | 急上昇ランキング | 直近1ヶ月のスコア急上昇セラピスト |
| Q | 訪問ログ | 口コミ書いてなくても「行った」マーク |
