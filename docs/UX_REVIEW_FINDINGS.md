## Agent 1: 未登録初訪問（検索→閲覧→ロック体験）✅完了

### 致命的（離脱に直結）

1. **トップページ: サイドバーが全て空** → 「新着セラピスト」準備中、「おすすめ店舗」空、「新着口コミ」読み込み失敗。活気ゼロに見える → SSRで初期データ確保 or 空ならセクション非表示
2. **トップページ:「広告枠 300x200」プレースホルダーがそのまま表示** → 未完成サイト感 → セクション削除
3. **検索結果: サロン見つかっても「0件のセラピストが見つかりませんでした」が目立つ** → サロン結果ある時は「セラピスト0件」非表示 or 「サロンが見つかりました!」強調

### 重要（体験を大きく損なう）

4. **ヘッダー「エリア検索」が `/area/tokyo` にハードコード** → 全国ユーザーが東京に飛ばされる → `/area` に変更
5. **ランキング: 全員92点・1件で差がない** → ランキングとして機能していない → 口コミ件数重み付け+エリアフィルター追加
6. **ヒーロー「6 タイプ分類」がハードコードで実際は8タイプ** → 数字不一致 → 「8」に修正
7. **VIP限定「サービスレベル検索」が未登録ユーザーに丸見え** → SKR/HRボタンが初訪問者に見える → 未ログインは非表示 or ブラー
8. **サロンページ: セラピスト一覧にページネーションなし** → 数十〜百人でページ超長い → 「もっと見る」追加

### 改善推奨（あると良い）

9. 検索バー2つ（ヘッダー+ヒーロー）の重複整理
10. エリアカードの「SV」表示 → 「人気度」に言い換え or 非表示
11. ランキングにエリアフィルター追加
12. セラピストページ「口コミ3件投稿で相性ラベル」→ 未登録時は「会員登録して〜」に変更
13. 口コミ1件のセラピストでロック体験なし → 一部をロックして導線確保
14. トップページ各セクションのクライアントfetch → SSR化
15. ヘッダーauth状態ロード中のちらつき対策

## Agent 3: リピーター検索体験（standard会員・再訪シナリオ）✅完了

### 致命的（離脱に直結）

1. **セラピスト名検索が機能しない** → `/api/therapists` にnameパラメータなし。228,816人のDBから最新50件をfetchしてクライアントフィルターのみ。名前で探しに来たリピーターが見つけられない → APIにname検索パラメータ追加（ilike `%name%`）
2. **検索ページネーションなし** → 50件固定で「もっと見る」なし。エリア検索でも50件超のセラピストは表示不可 → ページネーション or 無限スクロール追加
3. **おすすめ店舗がエリア非連動** → 大阪のエリアページなのにサイドバーに東京の店舗が表示される → prefecture_idでフィルター

### 重要（体験を大きく損なう）

4. **ランキングに口コミ1件で上位表示** → 1件92点で全員横並び。最低口コミ数の閾値なし → 最低3件以上でランキング対象に
5. **料金プランページに「ご利用中」表示なし** → standard会員がpricingページを見ても自分のプランがわからない → 現プランにバッジ表示
6. **お気に入り空状態のフォールバックが弱い** → 「お気に入りはありません」のみ → 「検索して気になるセラピストをお気に入りに追加しましょう」+ 検索CTA

### 改善推奨（あると良い）

7. サイドバーおすすめ店舗をユーザーの閲覧履歴/エリアに連動
8. リピーター向けに「前回見たセラピスト」セクション追加
9. 検索結果の並び替え（口コミ数順・評価順・新着順）
